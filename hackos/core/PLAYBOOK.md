## Purpose

The Playbook explains how to operate HackOS during a project.

Unlike the Kernel, the Playbook does not define framework behavior.

It describes recommended operational usage.

---

## Operating Philosophy

HackOS should always

Understand

before

Planning.

Plan

before

Implementation.

Validate

before

Submission.

## Step 5

---

Run Review.

Run Judge.

## Step 7

Prepare Demo.

Not feature count.

## Best Practices

Prefer

Small iterations.

Frequent validation.

Continuous runtime updates.

Evidence-based decisions.

Reliable demonstrations.

---

## Anti-Patterns

Avoid

Feature creep.

Premature optimization.

Large unvalidated implementations.

Ignoring Runtime Memory.

Bypassing the Orchestrator.

Skipping Review.
